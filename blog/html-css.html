CSS positioning

I'm going to tell you all about the 5 ways to position elements with CSS.

22nd April 2021 | Sprint 1

Static

This is the default position, unless stated HTML will assign this positioning to the element. It follows the document flow of the HTML code. So the first element in the code will come first, then the second etc.

Relative

Similar to static in that if follows the document flow, however top, right, bottom and left (TRBL) properties are made available to move the element relative to its original position. The element’s original position is anchored into the document flow but the element is technically removed from the flow and therefore doesn’t impact the position of any surrounding elements. For this reason TRBL properties are not commonly used with relative positioning as it may mean that other elements need to be moved to avoid overlap - this can get messy.

Absolute

Absolute positioning also has TRBL properties available to move the element - and it is recommended to use these properties with this positioning style. Where relative positioning moves relative to itself, absolute moves relative to its parent element. The parent element has to have a positioning value other than static for it to work otherwise it will become relative to the html element.

Fixed

Similar to absolute positioning however it will always move relative to the html element i.e. the entire page. As a result it is not affected by scrolling. It will stay in the same position on the screen - like my nav bar ;-)

Sticky

This one is a newer addition to CSS properties (consider it a love child of relative and fixed), it's only compatible with about 90% of browsers which makes it sound newer than the others - so use with caution! It allows elements to behave dynamically when scrolling the page. In essence this style is initially relative but as you scroll it will become fixed when it reaches a TRBL value.
